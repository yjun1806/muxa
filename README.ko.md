에이전트에게 일을 맡기는 곳은 터미널인데, 뭘 만들었는지 확인하려면 결국 에디터를 켜고 Git 클라이언트를 켠다. README 하나 읽자고 창을 옮기고, 다이어그램을 보려고 또 옮긴다. **muxa**는 파일 탐색기·마크다운/코드 뷰어·diff를 터미널과 같은 창에 둔다. 문서는 렌더된 상태로 보이고, 변경은 그 문서 위에 표시된다.

여러 세션을 나눠 띄우는 일도 한다. 이름은 **mux + a(gent)** — 터미널 멀티플렉서에 "보는 눈"을 더했다. 터미널 코어는 **[ghostty](https://ghostty.org)를 그대로 임베딩**했다. 웹뷰로 흉내 낸 게 아니라 GPU로 그리는 실제 터미널이 들어 있다.

> **개인 프로젝트를 있는 그대로 공개한 것입니다.** 상용 제품이 아니고 지원도 없으며, 기능 업데이트는 비정기적이고 아예 멈출 수도 있습니다. 배포용 `.dmg`나 공증(notarization)도 없고, **필요하면 직접 클론해서 빌드**해 씁니다. → [빌드해서 실행](#빌드해서-실행)
>
> **현재 Claude Code 전용입니다.** 알림·상태 표시의 1차 소스가 Claude Code 훅이라 지금은 Claude Code에 맞춰져 있습니다. 다른 에이전트 지원은 로드맵에 있습니다. → [지원 에이전트](#지원-에이전트)

## 왜 muxa?

터미널에서 Claude Code로 에이전트에게 작업을 맡기게 된 뒤로도, 결과를 볼 때만은 다른 앱이 필요했다. Git GUI를 띄우고 VSCode를 띄우고, 확인이 끝나면 다시 터미널로 돌아왔다. 그 왕복이 불편해서 만들었다.

그래서 muxa가 갈고닦는 "터미널 품질"은 렌더러 충실도가 아니다. **읽는 경험** — 문서를 문서로 보여주는 것, 변경을 원래 모양 위에 표시하는 것, 여러 세션의 상태를 한 목록에 모으는 것이다.

### 많이 열어도 어지럽지 않다

뷰어가 편하면 파일을 많이 연다. 그러면 탭이 스무 개가 되고 결국 안 쓰게 된다. muxa는 파일을 열 때 종류를 보고 알아서 묶는다. **문서·HTML·코드·미디어·변경** 다섯 갈래뿐이라 상위 탭이 그 이상 늘지 않고, 각 그룹 안에서 서브탭으로 오간다. 그룹 탭은 **항상 같은 순서**라 위치가 바뀌지 않고, 같은 파일을 다시 열면 **이미 있는 서브탭으로 이동**해 중복이 안 생긴다. 서브탭마다 스크롤 위치가 따로 살아 있다. `⌘W`는 서브탭 하나만 닫는다. 터미널은 이 묶음에 들어가지 않는다 — 그룹은 뷰어 전용이다.

### 변경을 문서 위에서 본다

에이전트는 코드만 고치지 않는다. README·설계 문서·작업 기록을 계속 다시 쓴다. 이런 파일을 `+`·`−` 패치로 읽으면 표가 깨지고 코드 블록이 뒤섞인다. muxa는 **마크다운을 렌더한 다음 그 위에 변경을 칠하므로**, 표는 표로 남고 셀 하나가 바뀌면 그 셀만 강조된다. 다이어그램은 도형으로 그려지고, 여러 번 고친 문단도 색이 겹쳐 검게 뭉개지지 않는다. 코드는 줄 번호와 미니맵이 붙은 일반 diff다.

바뀐 줄에 남긴 **코멘트는 터미널로 전달된다.** 검토 의견이 그대로 다음 지시가 된다. 반영하지 않을 변경은 파일 단위나 헝크 단위로 되돌린다. 커밋과 푸시는 터미널에서 한다.

### 워크트리를 넘어 세션이 따라온다

에이전트에게 브랜치를 맡기면 워크트리가 늘어난다. muxa는 저장소를 감시하다가 새 폴더가 생기면 **즉시 인식하고** 프로젝트로 추가할지 묻는다(주기적으로 훑는 게 아니라 변화를 직접 감시한다). 추가된 워크트리는 사이드바에 나란히 서고 클릭 한 번으로 전환한다 — 각자 자기 탭·분할·서비스를 따로 갖는다. 새 워크트리를 **앱 안에서 직접 만들 수도** 있다(브랜치를 고르면 폴더 생성과 git 설정까지 처리한다).

워크트리 안에서 도는 세션이 감지되면 그 칸 위에 **이동 제안**이 뜬다. 수락하면 작업이 끊기지 않고 옮겨간다. 단, 이식할 수 있는 건 앱 바깥에 사는 **∞ 지속 세션뿐**이라, 일반 터미널은 "가서 보기"만 준다 — 안 되는 것을 되는 것처럼 표시하지 않는다. 폴더가 삭제되어도 탭을 자동으로 닫지 않는다. 안에서 실행 중인 작업이 있을 수 있어 취소선으로 표시만 하고 판단은 사용자에게 맡긴다.

### 앱을 닫아도 에이전트가 죽지 않는다

터미널 세션을 앱 바깥의 **tmux**에 맡긴다. 앱을 종료해도, 강제 종료해도, 업데이트로 재시작해도 돌던 작업이 그대로 이어진다 — 레이아웃만 복원하고 프로세스는 죽이는 흔한 방식과 다르다. tmux가 설치돼 있으면 **새 탭이 기본으로 지속 세션(∞)** 이다. 기본을 이렇게 정한 건 손실이 비대칭이기 때문이다. 불필요한 세션은 낭비로 끝나지만, 지속이 아닌 탭의 에이전트는 앱을 닫는 순간 되돌릴 수 없이 죽는다. 기본은 살아남는 쪽이어야 한다 — 필요 없으면 탭을 닫으면 그만이다. tmux가 없으면 일반 셸로 열리고 ∞ 표시도 붙이지 않는다. 되는 척하지 않는다.

### 알림 — 승인 요청과 작업 완료

에이전트가 승인을 요청하거나 작업을 마치면 macOS 알림이 뜬다. 자리를 비운 사이의 알림은 인박스에 쌓이고, **알림 본문에는 에이전트가 마지막으로 한 말이 그대로** 들어간다(정형문이 아니라 실제 결과 요약). 항목을 고르면 해당 칸으로 간다. 아이콘은 사이드바·탭과 **같은 상태 글리프**라 다시 배울 게 없다.

알림을 켜려면 Claude Code가 muxa에 신호를 보내도록 **한 번만** 연결한다(인박스 아래 버튼, 기존 설정은 백업). 연결하면 승인 대기와 완료를 정확히 구분하고, 연결 전에는 화면이 잠잠하면 유휴로 **추정**한다 — 앱이 추정이라는 걸 숨기지 않는다.

### 분할과 탭 — 칸마다 다른 에이전트

화면을 원하는 비율로 나누고 각 칸에서 다른 세션을 돌린다. 포커스는 **밝기로만** 표시한다(선택 안 된 칸이 어두워질 뿐 테두리는 안 생긴다). **테두리는 알림이 독점**하므로 — 승인 대기는 깜빡이고 완료는 고정 — 칸이 넷이어도 확인이 필요한 곳을 즉시 구분한다. 칸마다 탭바가 따로 있고, 탭 왼쪽 표시가 그 탭 안 에이전트의 상태다. 제목 앞 **∞ 는 지속 세션** 표시. `⌘F`로 스크롤백 전체를 검색한다.

### 서비스 — 탭과 독립적으로 실행한다

## 빌드해서 실행

배포 바이너리를 두지 않으므로 **소스에서 직접 빌드**한다. zig는 필요 없다 — 터미널 코어는 소스가 아니라 prebuilt를 내려받는다.

**필요한 것**: macOS 14+ · Xcode Command Line Tools(`xcode-select --install`)

```sh
git clone https://github.com/yjun1806/muxa.git
cd muxa
./scripts/bootstrap.sh   # GhosttyKit.xcframework 설치 (최초 1회, SHA 고정 다운로드)
make app                 # .app 번들로 빌드·실행 (아이콘·시스템 알림 정상)
```

`make app`은 아이콘·번들 id가 붙은 정식 앱을 띄운다. `/Applications`에 설치하려면 `make install`(release 빌드)을 쓴다. 자세한 세팅·업그레이드 절차는 **[docs/SETUP.md](docs/SETUP.md)**.

### 필요한 외부 도구

| 도구 | 용도 | |
|---|---|---|
| `git` | diff·히스토리·워크트리 | 필수 |
| `tmux` | 지속 세션·서비스·스크립트 (없으면 앱이 설치 명령을 안내) | 선택 |
| `gh` | PR 배지 | 선택 |

### 첫 실행 권한

- **알림** — 에이전트가 나를 기다릴 때 알려준다(거부해도 앱 안 인박스에는 쌓인다).
- **폴더 접근**(문서·데스크탑·다운로드) — 그 폴더의 프로젝트를 열고 git 상태를 읽는다. 거부하면 파일 트리가 빈다.

알림을 받으려면 Claude Code가 muxa에 신호를 보내도록 한 번 연결해야 한다. 인박스 아래 **설치** 버튼을 누르면 되고, 기존 설정은 백업된다. 연결하지 않아도 쓸 수 있지만 상태 표시가 화면 변화를 보고 추정하는 방식으로 내려간다.

**설정**은 앱 메뉴 › `설정 파일 열기…`(`⌘,`) — `~/.config/muxa/config`에 주석 달린 기본본이 생기고, 저장하면 즉시 반영된다.
**단축키**는 메뉴바 `명령` 메뉴에 전부 있다.

## 하지 않는 것

muxa는 **여러 에이전트를 동시에 다루는 로컬 작업 환경** 하나에 집중한다. 아래는 설계상 범위 밖이거나 아직 지원하지 않는 항목이다.

- **코드를 편집하지 않는다.** 모든 뷰어는 읽기 전용이다. 코드는 에이전트가 고치고, 사람은 읽고 판단한다. 손대야 하면 에디터를 따로 연다.
- **Claude Code만 지원한다.** 상태 표시와 알림이 Claude Code에 맞춰져 있다. 다른 에이전트도 터미널에서 돌릴 수는 있지만 상태가 정밀하게 잡히지 않는다.
- **macOS 전용이다.** macOS 14 이상에서만 동작한다. Windows·Linux 계획은 없다.
- **원격 세션의 상태는 잡지 못한다.** 터미널이니 `ssh`로 다른 서버에 붙는 것 자체는 된다. 다만 알림·상태 감지는 로컬 훅과 소켓에 기대므로, 원격에서 도는 에이전트의 대기·완료는 표시되지 않는다. 폰에서 확인하는 기능도 없다.

<br />

## 지원 에이전트

지금은 **Claude Code 전용**이다. 알림·상태 표시의 1차 소스가 Claude Code 훅(`muxa-notify`)이라, 완결된 상태 판정(작업중·대기·완료·유휴)이 Claude Code에 맞춰져 있다. 다른 CLI 에이전트도 터미널로 띄워 쓸 수는 있지만 정밀한 알림은 아직 Claude Code에서만 완전하다.

다른 에이전트(Codex·Gemini 등) 지원은 **로드맵**에 있다.
