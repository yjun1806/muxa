# DESIGN.md — muxa UI 디자인 시스템

muxa의 **시각 언어**. 색·타이포·간격·컴포넌트·레이아웃 규칙이 여기 다 있다.
UI를 만들거나 고칠 때는 이 파일 하나만 읽으면 된다.

> **문서 분담** — 여기는 **어떻게 보이나**.
> **왜 이렇게 만들었나**(결정 로그·아키텍처·서브시스템)는 [ARCHITECTURE.md](ARCHITECTURE.md).
> 현재 상태·다음 할 일은 [STATUS.md](STATUS.md).

**코드의 SSOT** — 이 문서의 값은 아래 파일들을 서술한 것이다. 값을 바꾸려면 **코드를 먼저** 바꾼다.
| 무엇 | 파일 |
|---|---|
| 색 | `macos/Sources/muxa/Palette.swift` |
| 간격·반경·행 높이·모션 | `macos/Sources/muxa/Design/Tokens.swift` |

## 1. 성격

**터미널 앱이다.** 콘텐츠(터미널·코드·diff)가 주인공이고 크롬은 조용해야 한다.

- **밀도 우선.** 행 높이 22~28pt, 본문 11~12pt. 여백으로 숨 쉬게 하는 앱이 아니다.
- **색은 아껴 쓴다.** 크롬은 무채색, 색은 **의미가 있을 때만**(포커스·상태·git).
- **하드코딩 금지.** 색·간격·크기는 전부 토큰. 같은 hex를 두 곳에 쓰지 않는다.
- **라이트/다크 동시 지원.** 모든 색은 동적 `NSColor`라 자동 전환된다.

## 2. 색

### 브랜드 — 키 컬러는 **teal** (앱 아이콘 배경색)

강조·포커스·선택·CTA는 **전부 이 키에서 파생**한다.

| 토큰 | hex | 용도 |
|---|---|---|
| `Brand.s50`~`s300` | `F0FDFA` `CCFBF1` `99F6E4` `5EEAD4` | 옅은 틴트(배경·hover) |
| **`Brand.key` (= s400)** | **`2DD4BF`** | **키 컬러 — 앱 아이콘 배경** |
| `Brand.s500`~`s900` | `14B8A6` `0D9488` `0F766E` `115E59` `134E4A` | 진한 강조(라이트 대비·음영) |
| `Brand.gradTop` / `gradBottom` | `35DECA` / `27C4B1` | 아이콘 배경 그라디언트 |

**브랜드 semantic** (라이트 · 다크):

| 토큰 | 라이트 · 다크 | 용도 |
|---|---|---|
| `brand` (= `borderFocus`) | s600 · s400 | **포커스·활성·선택·CTA — 강조의 단일 출처** |
| `brandSubtle` | s100 · s800 | 옅은 강조 배경·선택 하이라이트 |
| `brandHover` | s700 · s300 | hover 강조 |
| `onBrand` | `0A2E2A` (딥 차콜) | 키 컬러 위 전경(텍스트·아이콘) |

### 표면·텍스트 (라이트 · 다크)

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `bg` | `FFFFFF` | `1B1B1D` | 콘텐츠·칸 배경 |
| `panel` | `F3F4F6` | `252528` | **크롬**(상단바·사이드바·푸터·패널 헤더) — 한 덩어리 회색 |
| `border` | `E2E5E9` | `38383C` | 경계선 |
| `fg` | `1F2937` | `E4E4E7` | 본문 텍스트 |
| `muted` | `6B7280` | `8A8A90` | 보조 텍스트·아이콘 |
| `mutedHover` | `1F2937` | `E4E4E7` | 보조 요소 hover(진해진다) |
| `btnHover` / `btnActive` | `E5E7EB` / … | `37373B` / … | 칩·버튼 배경 |

**크롬은 한 덩어리다.** 상단바·사이드바·푸터는 같은 `panel` 색이고 서로 **구분선을 긋지 않는다** —
"창 전체가 하나의 틀"로 읽혀야 한다. 콘텐츠는 그 위에 **카드**로 떠 있고(모서리 + 테두리), 층은 그것으로 드러난다.

### 기능색 — 브랜드와 별개, 의미 고정

관례적 의미색이라 키 컬러에 종속시키지 않는다.

| 의미 | 토큰 | 라이트 · 다크 |
|---|---|---|
| git added / **서비스 실행 중** | `gitAdded` / `serviceRunning` | `1A7F37` · `3FB950` (초록) |
| git modified | `gitModified` | `B08800` · `E2B341` (노랑) |
| git deleted / **에러·파괴적 동작** / **서비스 종료** | `gitDeleted` / `danger` / `serviceExited` | `CF222E` · `F85149` (빨강) |
| git renamed | `gitRenamed` | `0969DA` · `58A6FF` (파랑) |
| git conflict | `gitConflict` | `BC4C00` · `DB6D28` |
| **에이전트 활동·주의 환기** | `borderActivity` | `F59E0B` · `FBBF24` (주황) |
| PR merged | `prMerged` | `8250DF` · `A371F7` (보라) |

**색만으로 구분하지 않는다.** 상태는 색 + **글리프**로 함께 말한다(색맹 안전).
예: 서비스가 죽으면 초록 점(`circle.fill`)이 빨간 경고(`exclamationmark.triangle.fill`)로 **모양까지** 바뀐다.

## 3. 타이포

시스템 폰트(SF). 터미널 앱이라 **작고 촘촘하다**. 크기는 6단계뿐이고, 그 밖의 숫자는 쓰지 않는다.

| 스케일 | pt | 용도 |
|---|---|---|
| `nano` | 8 | DEV 배지 등 초소형 |
| `micro` | 9 | 아이콘, 상태 점 |
| `caption` | 10 | 보조 설명, 꼬리표 |
| `label` | 11 | **기본 UI 텍스트** (버튼·행·탭) |
| `body` | 12 | 본문 |
| `title` | 13 | 워드마크, 섹션 제목 |

- `.muxa(_ scale:weight:)` — 일반 텍스트
- `.muxaMono(_ scale:weight:)` — **수치·경로·명령·로그** (포트 `:3000`, exit code, 브랜치, 명령어)

**수치는 모노스페이스.** 자릿수가 바뀌어도 흔들리지 않아야 하고, 옆 텍스트와 성격이 다름이 드러난다.

## 4. 간격·반경·크기

**Space** (`Space.*`): `tight` 2 · `xs` 4 · `sm` 6 · `md` 8 · `panelInset` 10 · `lg` 12 · `xl` 16
- `sm`(6) = 기본 가로 배치 간격 — 대부분의 HStack
- `panelInset`(10) = 패널·도구줄 좌우 인셋

**Radius** (`Radius.*`): `sm` 4 · `md` 6(탭·버튼·칩) · `lg` 8(콘텐츠 카드)
> 터미널이 사는 창이라 **과하게 둥글리지 않는다.**

**RowHeight** (`RowHeight.*`): `tight` 22(칩·얕은 줄) · `row` 24(목록 행) · `toolbar` 28(도구줄·탭) · `bar` 30 · `header` 34 · `topBar` 38 · `hairline` 1

**Motion** (`Motion.*`): `fast` 0.12s · `medium` 0.18s (둘 다 `easeOut`)
> 크롬 전환은 빠르게. **칸 강조 테두리는 애니메이션하지 않는다** — 콘텐츠는 순간이동하는데 테두리만
> 미끄러지면 어긋나 보인다(색 전환만 부드럽게).

## 5. 레이아웃

### 영역 용어 (SSOT — 코드·문서·UI·커밋에서 이 이름으로 통일)

| 한국어 | English | 무엇 |
|--------|---------|------|
| **상단바** | Top Bar | 맨 위 줄 — 워크스페이스·프로젝트 탭, 알림 벨, 탐색기/Git 토글 |
| **사이드바** | Sidebar | 좌측 — 워크스페이스·프로젝트 전환 목록(●=백그라운드 활동) |
| **메인 영역** | Main Area | 중앙 전체 — **칸**들이 분할되어 들어가는 공간 |
| **칸** | Pane | 분할된 한 칸(탭들을 담는 단위). 좌우/상하 분할. Bonsplit pane |
| **탭** | Tab | 칸 안의 탭 — 터미널 탭 또는 **그룹 탭**(터미널·문서·HTML·코드·변경) |
| **서브탭** | Sub-tab | 그룹 탭 바로 아래 줄 — 그 그룹에 열린 개별 파일/커밋 |
| **도구 패널** | Tool Panel | 우측 — 파일 탐색기와 Git 패널이 토글로 번갈아 |
| **푸터** | Status Bar | 맨 아래 줄 — pwd·브랜치 · claude 사용량 칩 · 서비스 칩 |
| **서비스 도크** | Service Dock | 하단 **오버레이**(⌘J) — 좌: 서비스 목록, 우: 로그 |
| **콘텐츠 카드** | Content Card | 크롬 위에 떠 있는 판 — 터미널·패널이 그 안에 산다 |

**용어 구분·주의**
- **칸(Pane) ≠ 뷰어(Viewer)**: 칸은 분할 단위, 뷰어는 파일을 그리는 콘텐츠. "뷰"는 `TermView`(SwiftUI View)와 3중 충돌해 **쓰지 않는다**.
- **메인 영역 ≠ 워크스페이스**: VSCode 한국어판이 Workspace를 "작업 영역"으로 번역하므로 "작업 영역"을 쓰지 않는다.
- **도구 패널(우) ≠ 사이드바(좌)**: "사이드 패널"을 쓰지 않는다.
- 계층: **워크스페이스 › 프로젝트 › 칸 › 탭 › 서브탭**.

### 화면

```
┌──────┬──────────────────────────────┬───────────────┐
│ WS   │ ▶ agent  build  logs   [+]   │ EXPLORER      │  ← 상단바
│ ● A  │                              │ ▸ src/    M   │
│   B  │  $ claude                    │ ▾ docs/       │
│   C  │  ⏺ Writing report.md...      │    report.md ●│
│      │  █                           ├───────────────┤
│      │                              │ GIT  ⎇ main   │
│  [+] │                              │ M src/api.ts  │
│      ├──────────────────────────────┴───────────────┤
│      │ 서비스 │ web :3000                     ↻ 🗑 ✕ │  ← 서비스 도크(⌘J, 오버레이)
├──────┴────────────────────────────────────────────  ┤
│ 📁 ~/repo  ⎇ main          [✳ 5h 9%]  [● 2]         │  ← 푸터
└─────────────────────────────────────────────────────┘
```

**규칙**
- **분할은 두 층.** 바깥 크롬(사이드바·도구 패널)은 **역할 고정 접이식 패널**, 안쪽 메인 영역만 **자유 재귀 분할**. 통합하지 않는다(복잡도 폭발).
- **모든 패널은 접힌다.** 다 접으면 그냥 터미널 앱이다.
- **사이드바는 오버레이**다 — 레이아웃엔 접힌 폭만 예약하고, hover peek로 펼쳐져도 콘텐츠가 밀리지 않는다.
- **서비스 도크도 오버레이**다 — 레이아웃을 차지하면 여닫을 때마다 ghostty 그리드가 리플로우되고, 그 비용이 "슬쩍 보고 닫기"를 막는다. 겹치면 리플로우가 0이다.
- **칸 강조 테두리는 카드 클립 *바깥*에서 그린다** — 안에서 그리면 카드 라운드에 깎인다. (`ContentCard`)

## 6. 컴포넌트

| 컴포넌트 | 무엇 | 규칙 |
|---|---|---|
| `IconButton` | 도구줄·행 끝의 작은 SF Symbol 버튼 | **14×14 고정.** 크기가 유연하면 HStack이 공간을 배분할 때 텍스트에 밀려 0폭으로 사라진다 |
| `Pill` | 배지(PR 번호·DEV·상태) | 배경은 전경색의 옅은 틴트(`opacity 0.14`) — 색 하나만 정하면 된다 |
| `Meter` | 가로 막대(사용량) | 값 + 상태색 |
| `ContentCard` | 콘텐츠 판 + 칸 강조 테두리 레이어 | 위 레이아웃 규칙 참조 |
| **칩** | 푸터의 누를 수 있는 알약(사용량·서비스) | 배경(`btnHover`) + hover/열림 상태로 "누를 수 있음"을 드러낸다. 항목 사이는 여백이 아니라 **얇은 세로선**(`VDivider`)으로 가른다 — 여백만으로는 붙어 읽힌다 |
| **팝오버** | 칩의 상세 | 칩이 "있나 없나"를 말하고, 팝오버가 "무엇이 왜"를 말한다 |

**아이콘**: SF Symbols. 파일 아이콘만 Material Icon Theme(`Resources/fileicons`).

## 7. 글쓰기 (UI 문구)

- **모르면 침묵한다.** 값을 지어내지 않는다 — 포트를 못 뽑았으면 아무것도 안 붙이고, pwd를 모르면 프로젝트 경로로 **폴백하지 않는다**(거짓말이 된다).
- **원인을 말한다.** "오류가 발생했습니다"가 아니라 "`web`이 exit 3으로 죽었습니다 — 클릭해 로그 보기".
- **다음 행동을 준다.** 빈 상태는 설명 + 버튼(예: "등록된 서비스가 없습니다" + [서비스 추가]).
- 한국어. 기술 용어·식별자는 원문 그대로(`exit 3`, `pnpm dev`, `HEAD`).
