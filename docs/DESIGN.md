# DESIGN.md — muxa UI 디자인 시스템

muxa의 **시각 언어**. 색·타이포·간격·컴포넌트·레이아웃 규칙이 여기 다 있다.
UI를 만들거나 고칠 때는 이 파일 하나만 읽으면 된다.

> **문서 분담** — 여기는 **어떻게 보이나**.
> **왜 이렇게 만들었나**(결정 로그·아키텍처·서브시스템)는 [ARCHITECTURE.md](ARCHITECTURE.md).
> 현재 상태·다음 할 일은 [STATUS.md](STATUS.md).

**코드의 SSOT** — 이 문서의 값은 아래 파일들을 서술한 것이다. 값을 바꾸려면 **코드를 먼저** 바꾼다.

|---|---|
| 색 | `macos/Sources/muxa/Palette.swift` |
| 간격·반경·행 높이·모션 | `macos/Sources/muxa/Design/Tokens.swift` |

## 1. 성격

**터미널 앱이다.** 콘텐츠(터미널·코드·diff)가 주인공이고 크롬은 조용해야 한다.

- **밀도 우선.** 행 높이 22~28pt, 본문 11~12pt. 여백으로 숨 쉬게 하는 앱이 아니다.
- **색은 아껴 쓴다.** 크롬은 무채색, 색은 **의미가 있을 때만**(포커스·상태·git).
- **하드코딩 금지.** 색·간격·크기는 전부 토큰. 같은 hex를 두 곳에 쓰지 않는다.
- **라이트/다크 동시 지원.** 모든 색은 동적 `NSColor`라 자동 전환된다.

## 2. 색

### 크롬은 완전 무채 웜 그레이, 색은 신호다 (색상 피로도 최적화)

8시간 응시하는 터미널에서 **채도 = 피로**다(잔상 = 면적×시간×채도). 게다가 ANSI 콘텐츠가 이미 화면의
유채색 예산을 다 쓴다. 그래서 화면의 색을 셋으로 나눈다:
- ① **배경·크롬 전부 = 완전 무채 웜 그레이**(채도 ≈0 → 피로 ≈0, 웜이라 차갑지 않다). zinc(청보라 H≈286)의
  차가움을 걷어내고 웜(H≈40 근처, 채도 최소)으로 돌렸다.
- ② **상태 점 = 상태색**(작업중 `work` 인디고 · 대기 `waiting` 로즈 · 완료 `done` 세이지 · git · 서비스) — 작은 점만, 조용한 배경 위에서 pop.
- ③ **`brand` = 선셋 버밀리언 — 강조 지점에만 조금씩**(포커스·선택 지시선·CTA·링크·아이콘). 면이 아니라
  **선·글리프로만**, 크롬 픽셀의 1% 미만. 그래서 가장 뜨거운 색인데도 눈이 안 지친다.

**포커스는 색이 아니라 밝기로 말한다**(D20) — 포커스 없는 칸에 `paneVeil`을 덮는다.
칸 **테두리**는 그 칸 에이전트의 **상태**(작업중·대기·완료)를 상태별 형태·모션으로 지속 표시한다(§4 칸 상태).

- **선택은 중립 채움이다.** 목록·행·트리의 선택 표시는 `btnActive`(무채). 브랜드색 wash 금지 — 버밀리언은
  그 위에 **얇은 서명선**으로만 얹는다(사이드바 선택 프로젝트 좌측 2.5pt).
- **강조는 색이 아니라 크기·굵기·간격으로도 만든다.** 임의 opacity 곱(`.opacity(0.6)`)으로 색을 흐리지 않는다.

### 브랜드 — 버밀리언은 강조에만, 상태색은 인디고·로즈·세이지

`brand`(옛 딥틸)가 UI 강조와 "작업 중" 상태를 겸하던 걸 **버밀리언 리브랜드**에서 갈랐다: 상태색을 UI 강조에서 분리해
**세 상태 전용색**으로 두고(작업중 인디고·대기 로즈·완료 세이지), 버밀리언은 UI 강조 전용이다.
작업중은 처음 틸이었으나 **완료 세이지·git 초록과 너무 가까워 인디고(쿨톤)로 재배정**했다 — "돌고 있다"가 확실히 갈린다.
세 값은 `Palette.StatusHex`가 **단일 출처(SSOT)** — `work/waiting/done` NSColor·사이드바 마크·칸 테두리·**탭 마크**(`TabStatusMapping`)가 전부 여기서 파생한다(드리프트 없음).
아이콘 색(`Palette.Brand`)은 여전히 **격리**한다 — Dock 전용이라 UI accent보다 진해도 되고, UI에서 이 스케일을 꺼내 쓰지 않는다.

### 표면·텍스트 (라이트 · 다크) — 완전 무채 웜 그레이

zinc(청보라)의 차가움을 걷어내고 채도 ≈0의 **웜** 그레이로. 다크 bg는 순검정을 피해 L\*≈11(halation 완화),
라이트 크롬은 순백 대신 웜 오프화이트. 명도 계단(ΔL* 다크 7.8·라이트 5.5, panel→hover→active 사다리)은 온도만 바꿔 계승.

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `bg` | `FCFBFA` | `1C1A18` | 콘텐츠·칸 배경. 라이트 웜 오프화이트, 다크 웜 차콜 |
| `panel` | `F0EEEB` | `2B2926` | **크롬**(상단바·사이드바·푸터·패널 헤더) — 한 덩어리 웜 그레이 |
| `border` | `DEDAD5` | `423F3A` | 경계선·카드 테두리 + **카드 안 패널↔터미널 분할선** |
| `lane` | `F7F5F3` | `23211E` | 사이드바 트리 **프로젝트 레인** — 자식 묶음이 앉는 옅은 면(`bg` 55%+`panel` 45%). 가이드선·들여쓰기를 대체한다(소속을 선이 아니라 형태로) |
| `fg` | `26231F` | `E9E5E0` | 본문 텍스트 — 순백/순검정 회피, 웜 잉크 |

**크롬은 한 덩어리다.** 상단바·사이드바·푸터는 같은 `panel` 색이고 서로 **구분선을 긋지 않는다** —
"창 전체가 하나의 틀"로 읽혀야 한다. 콘텐츠는 그 위에 **카드**로 떠 있다.

**층은 명도차 *하나로만* 만들지 않는다 — 카드 고도와 나눠 진다.**
크롬을 명도차로만 벌리면 크롬 자체가 도형으로 읽히고("조용한 크롬"이 아니다),
그렇다고 명도차를 0으로 되돌리면 근백색·근흑색 터미널과 크롬이 한 톤으로 뭉갠다.
그래서 `Elevation.Card`(그림자 + 다크 상단 인셋 하이라이트)가 층의 절반을 지고, 명도차는 **중간값**만 진다:

- 다크 크롬↔콘텐츠 ΔL\*: **7.8** (명도차만 쓰던 10.2와, 고도만 믿던 5.4의 사이)
- 라이트: **5.5** (7.4 ↔ 3.8의 사이)

**고도가 닿지 않는 경계가 있다 — 그래서 명도차를 0으로 못 내린다.**
**카드 *안*의 도구 패널(익스플로러·git)↔터미널**에는 카드 그림자가 닿지 않는다 — 거긴 `panel`과 `bg`가
`border` 선 하나를 사이에 두고 직접 맞닿는다. 그 경계에선 명도차와 `border`가 유일한 신호다.
`border` 다크를 `34343A`→`3E3E44`로 올린 이유다(bg 대비 1.39→1.62).

**고도는 자리를 비워줘야 닿는다.** 사이드바는 카드 *위에* 뜨는 **불투명 오버레이**다(peek가 콘텐츠를
밀지 않으려면 그래야 한다). 카드가 사이드바에 딱 붙으면 카드 그림자(왼쪽으로 ~3pt 번진다)가 그
불투명 면에 통째로 가려져 — **층이 가장 필요한 사이드바↔터미널 경계에서만 고도가 0**이 된다.
그래서 카드 왼쪽에 크롬 `Space.xs`(4pt)를 비운다(`ContentView.contentCard`). 그림자가 설 자리다.

**peek 사이드바는 스스로 뜬다.** 펼쳐진 사이드바는 카드보다 위 레이어라 카드 고도가 못 비춘다.
남는 신호가 1px 하선(다크 1.62:1)뿐이면 2단 트리가 터미널에 얹힌 것처럼 보인다 —
peek 중에만 오른쪽으로 그림자를 드리운다(`Elevation.Peek`). 도킹 상태엔 0이다(크롬끼리 이어지는
자리에 그늘이 지면 얼룩으로 보인다).

**`btnActive`의 다크 값은 자유롭지 않다.** 그건 목록 선택 채움이면서 동시에 **칸 탭바의 면**이다
(`BonsplitChrome.colors.tabBar`). 활성 탭(`bg`)이 면으로 떠오르려면 그 아래 바가 눌려 있어야 하는데,
`3C3C41`은 `bg` 대비 1.57:1로 그 지표(1.9:1)를 절반쯤 되돌린다. `47474C`는 1.86:1이고 r≈g의 무채라
zinc 원칙과도 충돌하지 않는다. `panel`을 올린 만큼 hover도 한 칸 올려 사다리를 유지한다
(L\* `panel` 17.7 → `btnHover` 22.3 → `btnActive` 30.3).

**베일은 약하게 — 단, 두 모드의 *결과*를 맞춰서.** 칸을 나란히 놓고 두 터미널을 대조하는 게 이 앱의
일상이라 안 보이는 칸을 만들면 분할의 의미가 없다. 그렇다고 **같은 알파를 쓰면 다크에서 사라진다** —
검정을 곱하면 절대 변화량이 바탕 밝기에 비례하기 때문이다. 다크 12%는 `1B1B1D`를 `18181A`로 만들 뿐이라
**ΔL\* 1.52 · 1.03:1**로 빈 칸(프롬프트만 있는 셸)에선 아무 말도 못 했다. 그래서 알파가 아니라 **ΔL\*를 맞춘다**:

| 모드 | 알파 | 결과 | ΔL\* | 비포커스 칸 글자 |
|---|---|---|---|---|
| 라이트 | 3% | `FFFFFF`→`F7F7F7` | 2.77 | 13.9:1 |
| 다크 | **22%** | `1B1B1D`→`151517` | **3.00** | **8.6:1** (AAA 7:1 통과) |

### 기능색 — 브랜드와 별개, 의미 고정

관례적 의미색이라 키 컬러에 종속시키지 않는다. AA(4.5:1) 미달이던 값은 수리했다.

| 의미 | 토큰 | 라이트 · 다크 |
|---|---|---|
| git added | `gitAdded` | `1A7F37` · `3FB950` (초록) — 에이전트 완료는 `done`(세이지)로 **분리**(위 상태색 표) |
| **서비스 실행 중** | `serviceRunning`(=`gitRenamed` 파랑) | `0969DA` · `58A6FF` — 완료 세이지와 한 행에 나란히 떠서 분리(파랑=live) |
| git modified | `gitModified` | `9A6700` · `E2B341` (노랑) |
| git deleted / **에러·파괴적 동작** / **서비스 종료** | `gitDeleted` / `danger` / `serviceExited` | `CF222E` · `FF7B72` (빨강) |
| git renamed | `gitRenamed` | `0969DA` · `58A6FF` (파랑) |
| git conflict | `gitConflict` | `BC4C00` · `DB6D28` |
| **안내 배너·입력 검증 강조** | `borderActivity` | `A16207` · `FBBF24` (앰버) — 재개 배너·서비스 추가 시트·사용량 칩 등. (예전 "칸 활동 플래시"에 쓰였으나 **플래시는 제거**됐다) |
| PR merged | `prMerged` | `8250DF` · `B392F9` (보라) |

**색만으로 구분하지 않는다.** 상태는 색 + **모양**(글리프·크기)으로 함께 말한다(색맹 안전).
예: 서비스가 죽으면 초록 점(`circle.fill`)이 빨간 경고(`exclamationmark.triangle.fill`)로 **모양까지** 바뀌고,
프로젝트 상태 점은 유휴(5pt) → 신호(6pt)로 **크기가** 바뀐다(`ProjectStatusStyle`).

**상태 어휘는 세 축이다 — 축끼리 글리프를 공유하지 않는다.**
① 에이전트(`StatusStyle`): ● 인디고 / ⏸ 로즈 / ✓ `checkmark` 세이지.
② 서비스 = 끝없는 프로세스(`ServiceStatusStyle`): **원형 play/stop 짝** — ▶ `play.circle.fill` 파랑 / ■ `stop.circle` 무채 / ⚠ 빨강.
③ 스크립트 = 끝있는 명령(`ScriptStatusStyle`): **사각형 가족**(탭 아이콘 `play.square`와 한 축) —
실행중 ⟳ `arrow.triangle.2.circlepath` 파랑(`serviceRunning` 재사용 — 의미가 같고 글리프가 이미 가른다) /
성공 `checkmark.square` **무채**(에이전트 완료 세이지 ✓와 가름) / 실패 `xmark.square` 빨강 /
결과 미상 `questionmark.square` 무채(⌘W·프레임 유실 — ✓를 지어내지 않는다).

## 3. 타이포

시스템 폰트(SF). 터미널 앱이라 **작고 촘촘하다**. 크기는 6단계뿐이고, 그 밖의 숫자는 쓰지 않는다.

| 스케일 | pt | 용도 |
|---|---|---|
| `nano` | 8 | DEV 배지 등 초소형 |
| `micro` | 9 | 아이콘, 상태 점 |
| `caption` | 10 | 보조 설명, 꼬리표 |
| `label` | 11 | **기본 UI 텍스트** (버튼·행·탭) |
| `body` | 12 | 본문 |
| `title` | 13 | 워드마크, 섹션 제목 |

- `.muxa(_ scale:weight:)` — 일반 텍스트
- `.muxaMono(_ scale:weight:)` — **수치·경로·명령·로그·식별자** (포트 `:3000`, exit code, 브랜치, 명령어)
- `.muxaLabel` + `.tracking(Tracking.label)` + `.textCase(.uppercase)` — **소섹션 머리글**
  (10pt semibold. 설정 패널·도크 섹션 머리글이 이것. 1pt 계단만으로는 위계가 안 서므로
  크기를 낮추는 대신 **굵기·자간·대문자**로 "이건 항목이 아니라 머리글"이라고 말한다.
  머리글이 아닌 곳에 쓰지 않는다. **사이드바 워크스페이스 이름은 여기서 뺐다** —
  한글엔 대문자가 없어 반쪽이던 규칙이라, orca식 그룹 헤더(일반 케이스 body semibold + 개수 배지)로 바꿨다.)

**수치·식별자는 모노스페이스.** 자릿수가 바뀌어도 흔들리지 않아야 하고, 옆 텍스트와 성격이 다름이 드러난다.
사이드바·브레드크럼의 **자체 경로를 가진 프로젝트**(워크트리 등) 이름도 모노다 — 경로가 곧 신원이다(`Project.usesMonoName`).

## 4. 간격·반경·크기

**Space** (`Space.*`): `tight` 2 · `xs` 4 · `sm` 6 · `md` 8 · `panelInset` 10 · `lg` 12 · `xl` 16
- `sm`(6) = 기본 가로 배치 간격 — 대부분의 HStack
- `panelInset`(10) = 패널·도구줄 좌우 인셋
- `groupGap`(10) = 트리에서 **그룹 사이** 여백 — **간격이 위계다**(행 간격이 균일하면 목록이 한 덩어리로 뭉갠다)
  (예전 `treeIndent`(16)는 폐기 — 프로젝트 소속은 들여쓰기가 아니라 **레인**(`lane` 면)이 그린다)
> 스케일 밖의 값이 필요하면 **산술(`xl - xs`)로 만들지 말고 단계를 추가한다.**

**Radius** (`Radius.*`): `sm` 6(컨트롤·칩·탭·목록 행·작은 오버레이) · `lg` 10(콘텐츠 카드) — 2단으로 통일
> 터미널이 사는 창이라 **과하게 둥글리지 않는다.**

**RowHeight** (`RowHeight.*`): `tight` 22(칩·얕은 줄·트리 머리글) · `row` 24(목록 행·프로젝트 행) · `toolbar` 28(도구줄) · `bar` 30 · `header` 34 · `topBar` 38 · `hairline` 1

**IconSize** (`IconSize.*`): `inlineMark` 13 · `mark` 16 · `control` 24 · `statusSlot` 12(트리 행의 좌측 정사각 슬롯 — 상태 점·디스클로저·✕가 이 폭을 공유해야 두 단의 텍스트가 같은 세로선에서 시작한다) · `dot` 6(신호) · `dotSmall` 5(유휴) · `dotOffset` 3

**SlimBar** (`SlimBar.*`): 슬림(14pt) 사이드바의 색 막대 — `width` 3/`widthActive` 4 · `height` 18/`heightActive` 24 · `radius` 2
> **강조는 색이 아니라 폭·높이가 한다.** 막대 색은 상태(주의·작업중·유휴)만 말한다.

**Sidebar** (`Sidebar.*`): `hInset` 6(슬림은 0) · `chipGap` 8 · `chipOffset(width:)` — 이름 칩이 사이드바 **바깥**에 떠야 해서 인셋과 간격이 한 식으로 묶인다.

**Elevation.Card**: 도킹된 콘텐츠 카드의 **상시 미세 고도** — 그림자(라이트 0.06 · 다크 0.34, radius 3, y 1) + **다크 전용 상단 1px 인셋 하이라이트**(0.05).
> 떠 있는 판(`Elevation.key*`/`ambient*`)과는 다른 계급이다. 다크 UI에서 고도는 그림자보다 이 한 줄이 만든다.

**Motion** (`Motion.*`): `fast` 0.12s · `medium` 0.18s (둘 다 `easeOut`)
> 크롬 전환은 빠르게. **칸 테두리의 위치는 애니메이션하지 않는다** — 콘텐츠는 순간이동하는데 테두리만
> 미끄러지면 어긋나 보인다(리사이즈·분할 시). 색 전환은 부드럽게, 모션(펄스·흐름·글로우)은 `TimelineView`
> 시간구동이라 이 "순간이동" 원칙과 충돌하지 않는다.

### 칸 상태 — 테두리가 에이전트 상태를 말한다 (형태 × 모션 2축)

칸 테두리는 그 칸 에이전트의 **상태**를 지속 표시한다. **색은 고정, 표현은 형태 × 모션 두 축**을 곱해 만들고
**상태별(작업중·대기·완료)로 각각** 설정한다(설정 패널 "칸 상태 표시" 아코디언 → `PaneIndicatorSettings`).

| 상태 | 색 | 기본 형태·모션 | 사이드바·탭 마크(통일) |
|---|---|---|---|
| **작업 중** (working) | 인디고 `work` | 상단 진행바 + 흐름 | 회전 스피너 |
| **입력 대기** (waiting) | 로즈 `waiting` | 전체 링 + 펄스 | ⏸ `pause.fill` 펄스 |
| **완료** (done) | 세이지 `done` | 전체 링(정적) | ✓ `checkmark`(맨 체크 — v2 시안에서 원을 뺐다) |
| **유휴** (idle) | — | **표시 없음** | 탭: 없음(타입 아이콘 폴백) · 사이드바: **작은 무채 점**(빈 슬롯은 오류처럼 보였다) |

- **형태 6종**: 전체 링 · 상단/하단 바 · 좌측 레일 · 브래킷 · 코너 배지. **모션 4종**: 없음 · 펄스 · 흐름(진행바 — 바 전용, 그 외엔 펄스로 내림) · 글로우(이너 글로우, 분할 시 옆 칸 안 침범). 두께·글로우 번짐·브래킷 여백·속도는 슬라이더.
- **한 어휘를 세 곳이 공유**한다 — 사이드바 점(`StatusMark`)·탭 좌측 마크(Bonsplit `Tab.status` ← `TabStatusMapping`)·칸 테두리(`PaneBorderShape`)가 색·모양·모션이 같다.
- **작업중도 지속 표시**한다(예전엔 "작업 중엔 조용히"라 안 그렸다 — 진행 중이면 테두리가 계속 켜져 있어야 한다).
- **완료는 포커스하면 해제**(끝난 걸 확인). 대기는 설정 조건부(`clearOnFocus`, 기본 유지). 작업중·유휴는 유지.
- **유휴 ≠ 대기**: 끝나고 프롬프트에 앉은 상태는 idle(무표시)이고, 권한·질문 대기만 waiting(⏸)이다.
- **포커스에 테두리를 쓰지 않는다**(D20) — 포커스는 `paneVeil`(밝기)로. 테두리는 상태 전용이라 신호가 안 겹친다.
- 옛 **활동 플래시**(앰버 순간 테두리)는 **제거**했다 — 상태 테두리가 이미 상태를 지속 표시하므로 중복·소음이었다.

**용어 구분·주의**
- **분리 창(Project Window) ≠ 백그라운드 세션(Detached Session)**: `detach`는 **tmux 세션**이 선점한 낱말이다(`DetachedStrip`·`DetachedPopover`). 창을 떼는 동작은 **분리(separate)**, 되돌리는 동작은 **재합치기(rejoin)**로만 쓴다.
- 분리된 프로젝트는 사이드바에서 **사라지지 않는다** — 이름 뒤에 `macwindow` 마이크로 글리프가 붙고(활성 채움은 없다), 클릭하면 그 창이 앞으로 온다.
- **칸(Pane) ≠ 뷰어(Viewer)**: 칸은 분할 단위, 뷰어는 파일을 그리는 콘텐츠. "뷰"는 `TermView`(SwiftUI View)와 3중 충돌해 **쓰지 않는다**.
- **메인 영역 ≠ 워크스페이스**: VSCode 한국어판이 Workspace를 "작업 영역"으로 번역하므로 "작업 영역"을 쓰지 않는다.
- **도구 패널(우) ≠ 사이드바(좌)**: "사이드 패널"을 쓰지 않는다.
- 계층: **워크스페이스 › 프로젝트 › 칸 › 탭 › 서브탭**.

**사이드바 트리 규칙** (v2 — orca 헤더 × 레인 × 목록형 에이전트)
- **주인공은 폴더가 아니라 에이전트다.** 워크스페이스 행은 **orca식 그룹 헤더**로 조용하다:
  **리포 아바타**(git remote → GitHub owner 아바타, `RepoRemote` — 비 GitHub·remote 없음이면 레이어
  글리프/이니셜 폴백. 정체성 마크라 원본 색 유지 — ClaudeMark와 같은 규칙) + 일반 케이스 body semibold +
  **개수 배지**(프로젝트 수, 접혀도 보인다) + 셰브론은 **오른쪽·hover에서만**(접히면 −90° 회전 ▼→▶).
  선행 디스클로저는 없다 — **행 전체가 접기/펼치기 토글**이다.
  워크스페이스는 활성이어도 **채우지 않는다**(활성은 이름 색으로만).
- **소속은 선이 아니라 면이 그린다.** 펼친 워크스페이스의 자식 묶음은 **프로젝트 레인**(`lane` —
  옅은 면, 모서리 `Radius.lg` = 안쪽 행 `sm` + 인셋 `xs` 동심원) 위에 앉는다. 세로 가이드선·들여쓰기 폐기 —
  좁은 사이드바에서 긴 워크트리 이름에 가로 공간을 벌어준다. 가로 구분선도 없다(그룹 사이는 `groupGap`).
- **상태는 두 축이다.** 에이전트 축은 **롤업 점 하나**(가장 센 신호, `projectLeadingTone`)로 요약하고 —
  **유휴는 작은 무채 점**(5pt muted — 빈 슬롯은 아이콘 없는 행이 텅 비어 오류처럼 보였다. 신호와는
  크기(5↔6pt)·색으로 갈린다). **롤업은 점, 살아있는 글리프(스피너·펄스·체크)는 펼친 에이전트 행만** —
  개요와 실체를 모양으로도 가른다. **빨강은 에이전트에 없다** — 서비스 죽음에만(서비스 요약이 맡는다).
  접힌 그룹·icon·slim은 자식 중 **가장 센 신호**를 롤업한다(`SidebarTree.rollup`).
- **펼치면 에이전트/탭 목록 — L1 목록 행.** 각 행이 프로젝트 행과 **같은 문법**(풀폭 hover 채움 ·
  `RowHeight.tight` · `Radius.sm`)을 쓰고, "지금 보고 있는 탭"(포커스 칸의 선택 탭)은 **선택 채움**(`btnActive`).
  행 구성: **상태 글리프(WHAT) ⟂ 타입 마크(WHO: Claude 마크·터미널·문서…)** + 제목 – 본문 + **우측 열**
  (그룹=서브탭 개수 · 대기=경과, `muxaMono` — working/done은 시각 미저장이라 침묵).
  **그룹 행은 종류가 이름이다**("문서·HTML·코드" — 탭 제목을 반복하지 않는다). 뷰어 탭은 유휴여도 남기고,
  유휴 터미널만 "유휴 N"으로 접는다(접기 행도 터미널 아이콘을 달아 WHO 열이 끊기지 않는다).
- **프로젝트 행에도 개수 배지 — 열린 탭 수**(터미널+뷰어, 스타일은 `CountBadge` 공용 — 워크스페이스
  헤더의 프로젝트 수와 같은 문법). 아직 안 연(lazy) 프로젝트는 복원 스냅샷을 센다(`projectTabCount` —
  조회가 PTY를 스폰하지 않는다). 0이면 숨긴다(빈 배지는 소음).
- **클릭 규칙.** 워크스페이스 행 클릭 = 그 하나만 펼침/접힘 토글(아코디언 아님 — 활성도 접힌다).
  프로젝트 이름 클릭 = **전환 + 펼침**(전환하며 닫히지 않게 — 이미 활성인 걸 다시 누를 때만 토글).
  프로젝트 셰브론(hover에서만, 펼침 중엔 옅게 유지)도 언제든 토글.
  에이전트 목록 펼침은 **기본 펼침·접은 것만 영속**(`collapsedAgentLists`) — 재시작해도 접힌 채로
  시작하지 않는다(워크스페이스 펼침 영속과 같은 원리, 방향만 반대).
- **에이전트 행은 프롬프트가 곧 이름이다(2줄).** 마지막 입력 프롬프트(`UserPromptSubmit` 훅)가 있으면
  첫 줄 제목으로 승격하고(일감 목록으로 읽히게), 아랫줄이 `탭 이름 – 상태/라이브 도구`를 말한다.
  잘린 전문·첨부 이미지는 **hover 카드**(`PromptHoverCard`)가 보여준다. 프롬프트가 없으면(일반 셸·
  훅 이전 세션) 현행 1줄로 폴백 — 안 되는 곳을 만들지 않는다.
- **⌘n 힌트·`+`·셰브론은 hover에서만.** 헤더가 상시 시끄러우면 프로젝트 행이 안 보인다.
- **프로젝트 ✕는 열려 있는 터미널이 하나도 없을 때만** hover 노출 — 터미널이 살아 있으면 실수 클릭
  한 번이 모든 세션을 죽인다. 닫기의 정식 경로는 **우클릭 메뉴 → 확인 시트**.
- **접힌 모드(icon 52 · slim 14)도 2단이다** — 이름이 사라졌을 뿐 트리는 같다. 프로젝트 항목이 없으면
  프로젝트 전환·닫기·워크트리 생성에 마우스로 닿을 방법이 하나도 없어진다(상단 프로젝트 탭바가 사라졌으므로).
  워크트리 생성은 워크스페이스 우클릭 메뉴(`ProjectAddMenu`)에도 실려 모든 모드에서 열린다.

**규칙**
- **분할은 두 층.** 바깥 크롬(사이드바·도구 패널)은 **역할 고정 접이식 패널**, 안쪽 메인 영역만 **자유 재귀 분할**. 통합하지 않는다(복잡도 폭발).
- **모든 패널은 접힌다.** 다 접으면 그냥 터미널 앱이다.
- **사이드바는 오버레이**다 — 레이아웃엔 접힌 폭만 예약하고, hover peek로 펼쳐져도 콘텐츠가 밀리지 않는다.
- **서비스 도크는 우측 도킹 패널**이다 — 탐색기·Git과 같은 층에서 본문을 밀어내고, 좌측 경계 드래그로 너비를 조절한다(영속). 여닫을 때 ghostty 그리드가 리플로우되지만(초기엔 이 비용 때문에 오버레이였다), 도구 패널과의 일관성·리사이즈를 위해 감수한다.
- **칸 강조 테두리는 카드 클립 *바깥*에서 그린다** — 안에서 그리면 카드 라운드에 깎인다. (`ContentCard`)

## 6. 컴포넌트

| 컴포넌트 | 무엇 | 규칙 |
|---|---|---|
| `IconButton` | 도구줄·행 끝의 작은 SF Symbol 버튼 | **14×14 고정.** 크기가 유연하면 HStack이 공간을 배분할 때 텍스트에 밀려 0폭으로 사라진다 |
| `Pill` | 배지(PR 번호·DEV·상태) | 배경은 전경색의 옅은 틴트(`opacity 0.14`) — 색 하나만 정하면 된다 |
| `Meter` | 가로 막대(사용량) | 값 + 상태색 |
| `ContentCard` | 콘텐츠 판 + 칸 강조 테두리 레이어 | 위 레이아웃 규칙 참조. 카드 고도는 `Elevation.Card`(뒤판 그림자 — ghostty 서피스에 `.shadow`를 걸지 않는다) |
| `ProjectStatusStyle` | 프로젝트·워크스페이스 상태의 **색 + 점 크기** | `ServiceStatusStyle`와 같은 자리·같은 패턴. 트리 행·롤업 점·슬림 막대가 공유하는 단일 출처 |
| `EmptyState` | 빈 상태 — 아이콘 · 제목 · `subtitle`(왜 비었나) · 액션 | 아이콘·제목 크기는 **여기서만** 정한다(호출부가 `.system(size:)`를 다시 쓰지 않는다). `compact`는 팝오버용 |
| `EmptyStateButton` | `EmptyState` 액션 슬롯의 표준 버튼 | 조판(크기·채움·반경)은 여기서만 — 호출부 재조판 금지. `prominent`(브랜드 채움) = 1급 CTA, **화면당 하나만** |
| `ServiceRow` | 서비스 한 줄 — 도크 목록과 팝오버가 **같은 행**을 쓴다 | 표식은 **글리프**(`ServiceStatusStyle.glyph`) — 색만으로 상태를 말하지 않는다. 명령 미리보기는 `truncationMode(.tail)`: 가운데를 접으면 명령의 꼬리(`&& curl … \| sh`)가 숨는다 |
| `ScriptStatusStyle` | 스크립트 축(제3 축)의 색·글리프·라벨 SSOT | `ServiceStatusStyle`와 같은 자리·같은 패턴 — 푸터 칩(`ScriptStrip`)·팝오버 행이 공유 |
| `ScriptRow` | 스크립트 한 줄(팝오버 목록) | `ServiceRow`와 같은 규칙 — 글리프 표식 + 명령 `truncationMode(.tail)` |
| `Breadcrumb` | 상단바의 "지금 어디에 있나" | **라벨이다(버튼 아님).** 여기에 클릭을 주면 전환 경로가 둘이 되고, 어느 쪽이 진짜인지 매번 고민하게 된다 |
| `SidebarNameChip` | 접힌 모드에서 hover한 항목의 이름 칩 | 사이드바 **바깥**에 뜬다(`Sidebar.chipOffset`). 히트테스트 끔 |
| `muxaTip` | **팝오버 툴팁** — NSToolTip 대체(hover 0.5s 후 앵커 아래 칩) | 지연·모양·위치를 앱이 소유. 탭바(Bonsplit `safeHelp`)가 `BonsplitTooltipHost`로 위임받아 쓴다. **전역 `NSInitialToolTipDelay` 해킹 금지**(모든 툴팁이 즉시 떠 거슬린다 — 그래서 걷어냈다) |
| `muxaHoverCard` | **hover 카드** — 툴팁의 리치 판(임의 SwiftUI 콘텐츠) | 지연·이탈·클릭 접기 규칙은 `muxaTip`과 동일(`TipHostView`·`TipWindow` 공용). 창은 표시 시점 크기로 고정 — **비동기 로드는 자리(placeholder)를 먼저 잡는다** |
| `PromptHoverCard` | 잘린 프롬프트 **전문 + 첨부 이미지** 카드 | 사이드바 에이전트 행·백그라운드 팝오버 공용. 이미지는 transcript 마지막 user 메시지에서(최대 3장 + "+N"), 로드 전엔 무채 자리 |
| `PromptImageChip` | 첨부 이미지 칩(`photo` 글리프 + 개수) | "hover하면 보인다"의 어포던스 — 무채·개수만. 카드와 세트로 쓴다 |
| **칩** | 푸터의 누를 수 있는 알약(사용량·서비스·스크립트·백그라운드) | 배경(`btnHover`) + hover/열림 상태로 "누를 수 있음"을 드러낸다. 항목 사이는 여백이 아니라 **얇은 세로선**(`VDivider`)으로 가른다 — 여백만으로는 붙어 읽힌다. 스크립트·백그라운드 칩은 **0개면 숨김**(있을 때만 의미) — 서비스 칩만 상시(기능의 유일한 발견 지점) |
| **팝오버** | 칩의 상세 | 칩이 "있나 없나"를 말하고, 팝오버가 "무엇이 왜"를 말한다 |

**아이콘**: SF Symbols. 파일 아이콘만 Material Icon Theme(`Resources/fileicons`).

## 7. 글쓰기 (UI 문구)

- **모르면 침묵한다.** 값을 지어내지 않는다 — 포트를 못 뽑았으면 아무것도 안 붙이고, 에이전트가 뭘 하는지 훅 없이 추정만 될 땐 "무엇을"까지 말하지 않는다("작업 중"까지만).
- **원인을 말한다.** "오류가 발생했습니다"가 아니라 "`web`이 exit 3으로 죽었습니다 — 클릭해 로그 보기".
- **다음 행동을 준다.** 빈 상태는 설명 + 버튼(예: "등록된 서비스가 없습니다" + [서비스 추가]).
- 한국어. 기술 용어·식별자는 원문 그대로(`exit 3`, `pnpm dev`, `HEAD`).
