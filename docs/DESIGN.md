# DESIGN.md — muxa UI 디자인 시스템

muxa의 **시각 언어**. 색·타이포·간격·컴포넌트·레이아웃 규칙이 여기 다 있다.
UI를 만들거나 고칠 때는 이 파일 하나만 읽으면 된다.

> **문서 분담** — 여기는 **어떻게 보이나**.
> **왜 이렇게 만들었나**(결정 로그·아키텍처·서브시스템)는 [ARCHITECTURE.md](ARCHITECTURE.md).
> 현재 상태·다음 할 일은 [STATUS.md](STATUS.md).

**코드의 SSOT** — 이 문서의 값은 아래 파일들을 서술한 것이다. 값을 바꾸려면 **코드를 먼저** 바꾼다.

|---|---|
| 색 | `macos/Sources/muxa/Palette.swift` |
| 간격·반경·행 높이·모션 | `macos/Sources/muxa/Design/Tokens.swift` |

## 1. 성격

**터미널 앱이다.** 콘텐츠(터미널·코드·diff)가 주인공이고 크롬은 조용해야 한다.

- **밀도 우선.** 행 높이 22~28pt, 본문 11~12pt. 여백으로 숨 쉬게 하는 앱이 아니다.
- **색은 아껴 쓴다.** 크롬은 무채색, 색은 **의미가 있을 때만**(포커스·상태·git).
- **하드코딩 금지.** 색·간격·크기는 전부 토큰. 같은 hex를 두 곳에 쓰지 않는다.
- **라이트/다크 동시 지원.** 모든 색은 동적 `NSColor`라 자동 전환된다.

## 2. 색

### 크롬은 무채, 색은 신호다

터미널 콘텐츠가 이미 ANSI 유채색으로 가득하다. 크롬까지 유채색이면 신호가 아니라 소음이다.
그래서 크롬의 회색은 중립 무채(zinc)로 통일하고, 유채색은 두 계급만 남긴다 —
① 포커스·활성(`brand`) ② 상태(`borderActivity`·git·서비스).
**포커스는 색이 아니라 밝기로 말한다**(D20) — 포커스 없는 칸에 `paneVeil`을 덮고,
테두리 채널은 **에이전트 알림(`borderActivity`)에만** 내준다. 테두리가 뜨면 그건 진짜 알림이다.

- **선택은 중립 채움이다.** 목록·행·트리의 선택 표시는 `btnActive`(무채). 브랜드색 wash·좌측 컬러 레일 금지.
- **강조는 색이 아니라 크기·굵기·간격으로도 만든다.** 임의 opacity 곱(`.opacity(0.6)`)으로 색을 흐리지 않는다.

### 브랜드 — **아이콘의 teal과 UI의 teal은 다른 색이다**

앱 아이콘 배경(`Brand.key` = `2DD4BF`)은 L\*78.5·고채도라 다크 크롬(`1B1B1D`) 위에서 대비 **9.24:1** —
포커스 링에 필요한 3:1의 세 배고, 보조 텍스트보다 15포인트 밝아 **크롬에서 가장 빛나는 물체**가 된다.
(VS Code `007ACC` · Zed `2472F2` · Linear `5E6AD2`는 전부 L\* 55~62다.)
그래서 아이콘 색은 `Palette.Brand`에 **격리**하고 — UI에서 이 스케일을 꺼내 쓰지 않는다 —
UI 강조는 채도를 내린 **딥틸** semantic 토큰만 쓴다.

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `Brand.key` / `gradTop` / `gradBottom` | `2DD4BF` / `35DECA` / `27C4B1` | (동일) | **앱 아이콘 전용.** 실제 드로잉은 `scripts/build-appicon/icon-gen.swift`(값 미러링) |
| `brand` | `0F766E` | `5FB8AB` | 강조 **텍스트·글리프·1급 CTA 채움 + 선택 탭 포커스 지시선** (라이트 5.47:1 · 다크 5.9:1 — AA) |
| `brandSubtle` | `E6F5F1` | `223834` | 옅은 강조 배경 틴트. **목록 선택에는 쓰지 않는다** |
| `brandHover` | `0C5F59` | `79C4B8` | hover 강조 |
| `onBrand` | `FFFFFF` | `0A2E2A` | **`brand` 채움 위 전경.** brand의 명도가 모드마다 뒤집히므로 전경도 뒤집힌다(5.52:1 · 6.20:1) |

> **`borderFocus`는 없앴다.** 한때 "포커스 링 전용"의 더 가라앉힌 딥틸(다크 `3B8A7F`)을 뒀지만,
> ① 유일한 소비자였던 **칸 포커스 링이 `paneVeil`로 대체됐고**(D20 — 포커스는 색이 아니라 밝기),
> ② 남은 포커스 강조인 **선택 탭 지시선은 `bg`가 아니라 탭바(`btnActive`) 위에 얹힌다** — 거기서
> `3B8A7F`는 **2.26:1**로 비텍스트 3:1에도 못 미치고 `brand`는 3.93:1로 통과한다.
> 맡을 자리가 없어진 토큰이라 지웠다. 포커스 강조가 필요하면 `brand`를 쓴다.
>
> **단, 지시선 색 하나로 포커스를 말하지 않는다.** 탭바(`btnActive`) 위에서 재면 —
> 라이트: 포커스 `brand` **3.77:1** vs 비포커스 `muted` **3.99:1**(L\* 44.5 vs 43.0 — 사실상 같은 밝기).
> 다크: 3.93:1 vs 3.22:1. 즉 **라이트에선 색 대비가 포커스를 못 가른다**(색상만 다르다).
> 그걸 메우는 건 색이 아닌 채널들이다 — **베일**(칸 전체, 위 표) · **두께**(2pt vs 1pt) ·
> **제목 굵기**(semibold vs regular, D22). 색맹 사용자도 이 셋으로 읽는다.
> (지시선 전용 토큰을 새로 두는 대신 이 셋을 쓴다: `brand`는 두 모드 다 비텍스트 3:1을 통과하고 —
> 이전 값 `0D9488`은 라이트에서 **2.54:1**로 미달이었다 — 그 위에 achromatic 신호를 얹는 편이 토큰 하나를
> 더 만드는 것보다 색맹 안전에 더 직접적이다.)

### 표면·텍스트 (라이트 · 다크) — 중립 zinc

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `bg` | `FFFFFF` | `1B1B1D` | 콘텐츠·칸 배경 |
| `panel` | `EFEFF1` | `2B2B2F` | **크롬**(상단바·사이드바·푸터·패널 헤더) — 한 덩어리 회색 |
| `border` | `DCDCE0` | `3E3E44` | 경계선·카드 테두리 + **카드 안 패널↔터미널 분할선**(고도가 못 닿는 유일한 신호) |
| `guide` | `CBCBD2` | `54545C` | 사이드바 2단 트리의 **세로 가이드선** — 프로젝트를 워크스페이스 아래로 묶는다. 1px가 panel 위에서 읽히게 border보다 살짝 진하다 |
| `fg` | `232326` | `E4E4E7` | 본문 텍스트 |
| `muted` | `65656B` | `98989E` | 보조 텍스트·아이콘 (다크 `8A8A90`은 3.82:1로 AA 탈락이었다) |
| `mutedHover` | `232326` | `E4E4E7` | 보조 요소 hover(진해진다) |
| `btnHover` | `E6E6E9` | `35353A` | hover 채움 |
| `btnActive` | `D6D6D9` | `47474C` | **선택·활성 채움 — 목록 선택의 단일 출처.** 동시에 **칸 탭바 배경**이라 다크 값이 묶여 있다(아래) |
| `paneVeil` | 검정 3% | 검정 **22%** | **포커스 없는 칸**을 덮는 베일 → "지금 입력이 어디로 가나"를 밝기로 말한다 |

**크롬은 한 덩어리다.** 상단바·사이드바·푸터는 같은 `panel` 색이고 서로 **구분선을 긋지 않는다** —
"창 전체가 하나의 틀"로 읽혀야 한다. 콘텐츠는 그 위에 **카드**로 떠 있다.

**층은 명도차 *하나로만* 만들지 않는다 — 카드 고도와 나눠 진다.**
크롬을 명도차로만 벌리면 크롬 자체가 도형으로 읽히고("조용한 크롬"이 아니다),
그렇다고 명도차를 0으로 되돌리면 근백색·근흑색 터미널과 크롬이 한 톤으로 뭉갠다.
그래서 `Elevation.Card`(그림자 + 다크 상단 인셋 하이라이트)가 층의 절반을 지고, 명도차는 **중간값**만 진다:

- 다크 크롬↔콘텐츠 ΔL\*: **7.8** (명도차만 쓰던 10.2와, 고도만 믿던 5.4의 사이)
- 라이트: **5.5** (7.4 ↔ 3.8의 사이)

**고도가 닿지 않는 경계가 있다 — 그래서 명도차를 0으로 못 내린다.**
**카드 *안*의 도구 패널(익스플로러·git)↔터미널**에는 카드 그림자가 닿지 않는다 — 거긴 `panel`과 `bg`가
`border` 선 하나를 사이에 두고 직접 맞닿는다. 그 경계에선 명도차와 `border`가 유일한 신호다.
`border` 다크를 `34343A`→`3E3E44`로 올린 이유다(bg 대비 1.39→1.62).

**고도는 자리를 비워줘야 닿는다.** 사이드바는 카드 *위에* 뜨는 **불투명 오버레이**다(peek가 콘텐츠를
밀지 않으려면 그래야 한다). 카드가 사이드바에 딱 붙으면 카드 그림자(왼쪽으로 ~3pt 번진다)가 그
불투명 면에 통째로 가려져 — **층이 가장 필요한 사이드바↔터미널 경계에서만 고도가 0**이 된다.
그래서 카드 왼쪽에 크롬 `Space.xs`(4pt)를 비운다(`ContentView.contentCard`). 그림자가 설 자리다.

**peek 사이드바는 스스로 뜬다.** 펼쳐진 사이드바는 카드보다 위 레이어라 카드 고도가 못 비춘다.
남는 신호가 1px 하선(다크 1.62:1)뿐이면 2단 트리가 터미널에 얹힌 것처럼 보인다 —
peek 중에만 오른쪽으로 그림자를 드리운다(`Elevation.Peek`). 도킹 상태엔 0이다(크롬끼리 이어지는
자리에 그늘이 지면 얼룩으로 보인다).

**`btnActive`의 다크 값은 자유롭지 않다.** 그건 목록 선택 채움이면서 동시에 **칸 탭바의 면**이다
(`BonsplitChrome.colors.tabBar`). 활성 탭(`bg`)이 면으로 떠오르려면 그 아래 바가 눌려 있어야 하는데,
`3C3C41`은 `bg` 대비 1.57:1로 그 지표(1.9:1)를 절반쯤 되돌린다. `47474C`는 1.86:1이고 r≈g의 무채라
zinc 원칙과도 충돌하지 않는다. `panel`을 올린 만큼 hover도 한 칸 올려 사다리를 유지한다
(L\* `panel` 17.7 → `btnHover` 22.3 → `btnActive` 30.3).

**베일은 약하게 — 단, 두 모드의 *결과*를 맞춰서.** 칸을 나란히 놓고 두 터미널을 대조하는 게 이 앱의
일상이라 안 보이는 칸을 만들면 분할의 의미가 없다. 그렇다고 **같은 알파를 쓰면 다크에서 사라진다** —
검정을 곱하면 절대 변화량이 바탕 밝기에 비례하기 때문이다. 다크 12%는 `1B1B1D`를 `18181A`로 만들 뿐이라
**ΔL\* 1.52 · 1.03:1**로 빈 칸(프롬프트만 있는 셸)에선 아무 말도 못 했다. 그래서 알파가 아니라 **ΔL\*를 맞춘다**:

| 모드 | 알파 | 결과 | ΔL\* | 비포커스 칸 글자 |
|---|---|---|---|---|
| 라이트 | 3% | `FFFFFF`→`F7F7F7` | 2.77 | 13.9:1 |
| 다크 | **22%** | `1B1B1D`→`151517` | **3.00** | **8.6:1** (AAA 7:1 통과) |

### 기능색 — 브랜드와 별개, 의미 고정

관례적 의미색이라 키 컬러에 종속시키지 않는다. AA(4.5:1) 미달이던 값은 수리했다.

| 의미 | 토큰 | 라이트 · 다크 |
|---|---|---|
| git added / **서비스 실행 중** | `gitAdded` / `serviceRunning` | `1A7F37` · `3FB950` (초록) |
| git modified | `gitModified` | `9A6700` · `E2B341` (노랑) |
| git deleted / **에러·파괴적 동작** / **서비스 종료** | `gitDeleted` / `danger` / `serviceExited` | `CF222E` · `FF7B72` (빨강) |
| git renamed | `gitRenamed` | `0969DA` · `58A6FF` (파랑) |
| git conflict | `gitConflict` | `BC4C00` · `DB6D28` |
| **에이전트 활동·주의 환기** | `borderActivity` | `B45309` · `FBBF24` (호박) — 라이트 `F59E0B`는 흰 배경 대비 2.15:1로 사실상 안 보였다 |
| PR merged | `prMerged` | `8250DF` · `B392F9` (보라) |

**색만으로 구분하지 않는다.** 상태는 색 + **모양**(글리프·크기)으로 함께 말한다(색맹 안전).
예: 서비스가 죽으면 초록 점(`circle.fill`)이 빨간 경고(`exclamationmark.triangle.fill`)로 **모양까지** 바뀌고,
프로젝트 상태 점은 유휴(5pt) → 신호(6pt)로 **크기가** 바뀐다(`ProjectStatusStyle`).

## 3. 타이포

시스템 폰트(SF). 터미널 앱이라 **작고 촘촘하다**. 크기는 6단계뿐이고, 그 밖의 숫자는 쓰지 않는다.

| 스케일 | pt | 용도 |
|---|---|---|
| `nano` | 8 | DEV 배지 등 초소형 |
| `micro` | 9 | 아이콘, 상태 점 |
| `caption` | 10 | 보조 설명, 꼬리표 |
| `label` | 11 | **기본 UI 텍스트** (버튼·행·탭) |
| `body` | 12 | 본문 |
| `title` | 13 | 워드마크, 섹션 제목 |

- `.muxa(_ scale:weight:)` — 일반 텍스트
- `.muxaMono(_ scale:weight:)` — **수치·경로·명령·로그·식별자** (포트 `:3000`, exit code, 브랜치, 명령어)
- `.muxaLabel` + `.tracking(Tracking.label)` + `.textCase(.uppercase)` — **소섹션 머리글**
  (10pt semibold. 사이드바의 워크스페이스 이름이 이것. 1pt 계단만으로는 위계가 안 서므로
  크기를 낮추는 대신 **굵기·자간·대문자**로 "이건 항목이 아니라 머리글"이라고 말한다.
  한글은 대문자 변환이 없으니 자간·굵기만으로도 라벨로 읽혀야 한다. 머리글이 아닌 곳에 쓰지 않는다.)

**수치·식별자는 모노스페이스.** 자릿수가 바뀌어도 흔들리지 않아야 하고, 옆 텍스트와 성격이 다름이 드러난다.
사이드바·브레드크럼의 **자체 경로를 가진 프로젝트**(워크트리 등) 이름도 모노다 — 경로가 곧 신원이다(`Project.usesMonoName`).

## 4. 간격·반경·크기

**Space** (`Space.*`): `tight` 2 · `xs` 4 · `sm` 6 · `md` 8 · `panelInset` 10 · `lg` 12 · `xl` 16
- `sm`(6) = 기본 가로 배치 간격 — 대부분의 HStack
- `panelInset`(10) = 패널·도구줄 좌우 인셋
- `treeIndent`(16) = 2단 트리의 자식(프로젝트) 들여쓰기
- `groupGap`(10) = 트리에서 **그룹 사이** 여백 — **간격이 위계다**(행 간격이 균일하면 목록이 한 덩어리로 뭉갠다)
> 스케일 밖의 값이 필요하면 **산술(`xl - xs`)로 만들지 말고 단계를 추가한다.**

**Radius** (`Radius.*`): `sm` 4 · `md` 6(탭·버튼·칩) · `lg` 8(콘텐츠 카드)
> 터미널이 사는 창이라 **과하게 둥글리지 않는다.**

**RowHeight** (`RowHeight.*`): `tight` 22(칩·얕은 줄·트리 머리글) · `row` 24(목록 행·프로젝트 행) · `toolbar` 28(도구줄) · `bar` 30 · `header` 34 · `topBar` 38 · `hairline` 1

**IconSize** (`IconSize.*`): `inlineMark` 13 · `mark` 16 · `control` 24 · `statusSlot` 12(트리 행의 좌측 정사각 슬롯 — 상태 점·디스클로저·✕가 이 폭을 공유해야 두 단의 텍스트가 같은 세로선에서 시작한다) · `dot` 6(신호) · `dotSmall` 5(유휴) · `dotOffset` 3

**SlimBar** (`SlimBar.*`): 슬림(14pt) 사이드바의 색 막대 — `width` 3/`widthActive` 4 · `height` 18/`heightActive` 24 · `radius` 2
> **강조는 색이 아니라 폭·높이가 한다.** 막대 색은 상태(주의·작업중·유휴)만 말한다.

**Sidebar** (`Sidebar.*`): `hInset` 6(슬림은 0) · `chipGap` 8 · `chipOffset(width:)` — 이름 칩이 사이드바 **바깥**에 떠야 해서 인셋과 간격이 한 식으로 묶인다.

**Elevation.Card**: 도킹된 콘텐츠 카드의 **상시 미세 고도** — 그림자(라이트 0.06 · 다크 0.34, radius 3, y 1) + **다크 전용 상단 1px 인셋 하이라이트**(0.05).
> 떠 있는 판(`Elevation.key*`/`ambient*`)과는 다른 계급이다. 다크 UI에서 고도는 그림자보다 이 한 줄이 만든다.

**Motion** (`Motion.*`): `fast` 0.12s · `medium` 0.18s (둘 다 `easeOut`)
> 크롬 전환은 빠르게. **칸 테두리는 애니메이션하지 않는다** — 콘텐츠는 순간이동하는데 테두리만
> 미끄러지면 어긋나 보인다(색 전환만 부드럽게).

### 칸 상태 — 테두리는 **알림 전용**이다

| 무엇을 말하나 | 무엇으로 | 왜 |
|---|---|---|
| **지금 입력이 어디로 가나** (포커스) | 포커스 **없는** 칸에 `paneVeil` + 활성 탭 카드(teal 하단선·아이콘·굵은 제목) | 밝기는 **주변시로 읽힌다** — 탭바를 안 봐도 어느 칸이 밝은지 안다. 선을 안 그으니 크롬이 조용하다 |
| **에이전트가 나를 기다린다 / 끝냈다** | 칸 **테두리** (`borderActivity` 주황 / `gitAdded` 초록) | — |
| 방금 활동이 있었다 | 칸 테두리 플래시 (주황, 켤 땐 빠르게·끌 땐 천천히) | — |

> **포커스에 테두리를 쓰지 않는다.** 그건 *상시* 켜지는 신호인데, 같은 테두리 채널을 에이전트 알림도 쓴다.
> 청록 테두리가 늘 깔려 있으면 **정작 나를 부르는 주황이 그 위에서 경쟁**해야 한다 — 강조가 강조를 잡아먹는다.
> 테두리를 비워 두면 **"테두리가 떴다 = 무슨 일이 났다"**가 성립한다. (→ ARCHITECTURE D20)

**용어 구분·주의**
- **분리 창(Project Window) ≠ 백그라운드 세션(Detached Session)**: `detach`는 **tmux 세션**이 선점한 낱말이다(`DetachedStrip`·`DetachedPopover`). 창을 떼는 동작은 **분리(separate)**, 되돌리는 동작은 **재합치기(rejoin)**로만 쓴다.
- 분리된 프로젝트는 사이드바에서 **사라지지 않는다** — 이름 뒤에 `macwindow` 마이크로 글리프가 붙고(활성 채움은 없다), 클릭하면 그 창이 앞으로 온다.
- **칸(Pane) ≠ 뷰어(Viewer)**: 칸은 분할 단위, 뷰어는 파일을 그리는 콘텐츠. "뷰"는 `TermView`(SwiftUI View)와 3중 충돌해 **쓰지 않는다**.
- **메인 영역 ≠ 워크스페이스**: VSCode 한국어판이 Workspace를 "작업 영역"으로 번역하므로 "작업 영역"을 쓰지 않는다.
- **도구 패널(우) ≠ 사이드바(좌)**: "사이드 패널"을 쓰지 않는다.
- 계층: **워크스페이스 › 프로젝트 › 칸 › 탭 › 서브탭**.

**사이드바 트리 규칙**
- **주인공은 폴더가 아니라 에이전트다.** 프로젝트 행(`RowHeight.row`)이 크고, 워크스페이스 행은
  소섹션 머리글(`RowHeight.tight` + `muxaLabel`)로 조용하다. 워크스페이스는 활성이어도 **채우지 않는다**.
- **워크스페이스↔프로젝트는 시각 어휘로 가른다**(색 아님): 워크스페이스 = **레이어 글리프**(`square.stack`) +
  대문자 라벨(=컨테이너), 프로젝트 = **상태 점** + 이름(=항목). 소속·경계는 **세로 가이드선**(`guide`)이
  그린다 — 가로 구분선은 쓰지 않는다(간격이 위계다: 그룹 사이는 `groupGap`).
- **상태는 색이 아니라 모양으로도 말한다**: 유휴 = 흐린 작은 점(5pt) · 작업중 = 브랜드 점(6pt) ·
  주의 = 호박 점(6pt). 접힌 그룹·icon·slim은 자식 중 **가장 센 신호**를 롤업한다(`SidebarTree.rollup`).
- **에이전트 큐 — 활동만 2줄로 확장.** 프로젝트 행은 working·waiting일 때만 이름 아래 **상태 라인**
  (`작업 중`/`입력 대기`, 색은 활동이 정한다)을 붙인다. 유휴는 1줄로 조용히(`AppState.projectActivityLine` —
  **이미 만들어진 스토어만** 본다, 트리를 그리는 것만으로 PTY를 스폰하지 않는다).
- **각 워크스페이스는 독립적으로 여닫힌다**(아코디언 아님 — 하나를 열어도 다른 건 안 접힌다). 디스클로저
  클릭 = 전환 없이 그 하나만 접기/펼치기(활성도 접을 수 있다), 행 클릭 = 전환(포커스한 곳은 펼쳐 보여주되 남은 안 접음).
- **⌘n 힌트·`+`는 hover에서만.** 헤더가 상시 시끄러우면 프로젝트 행이 안 보인다.
- **접힌 모드(icon 52 · slim 14)도 2단이다** — 이름이 사라졌을 뿐 트리는 같다. 프로젝트 항목이 없으면
  프로젝트 전환·닫기·워크트리 생성에 마우스로 닿을 방법이 하나도 없어진다(상단 프로젝트 탭바가 사라졌으므로).
  워크트리 생성은 워크스페이스 우클릭 메뉴(`ProjectAddMenu`)에도 실려 모든 모드에서 열린다.

**규칙**
- **분할은 두 층.** 바깥 크롬(사이드바·도구 패널)은 **역할 고정 접이식 패널**, 안쪽 메인 영역만 **자유 재귀 분할**. 통합하지 않는다(복잡도 폭발).
- **모든 패널은 접힌다.** 다 접으면 그냥 터미널 앱이다.
- **사이드바는 오버레이**다 — 레이아웃엔 접힌 폭만 예약하고, hover peek로 펼쳐져도 콘텐츠가 밀리지 않는다.
- **서비스 도크는 우측 도킹 패널**이다 — 탐색기·Git과 같은 층에서 본문을 밀어내고, 좌측 경계 드래그로 너비를 조절한다(영속). 여닫을 때 ghostty 그리드가 리플로우되지만(초기엔 이 비용 때문에 오버레이였다), 도구 패널과의 일관성·리사이즈를 위해 감수한다.
- **칸 강조 테두리는 카드 클립 *바깥*에서 그린다** — 안에서 그리면 카드 라운드에 깎인다. (`ContentCard`)

## 7. 글쓰기 (UI 문구)

- **모르면 침묵한다.** 값을 지어내지 않는다 — 포트를 못 뽑았으면 아무것도 안 붙이고, 에이전트가 뭘 하는지 훅 없이 추정만 될 땐 "무엇을"까지 말하지 않는다("작업 중"까지만).
- **원인을 말한다.** "오류가 발생했습니다"가 아니라 "`web`이 exit 3으로 죽었습니다 — 클릭해 로그 보기".
- **다음 행동을 준다.** 빈 상태는 설명 + 버튼(예: "등록된 서비스가 없습니다" + [서비스 추가]).
- 한국어. 기술 용어·식별자는 원문 그대로(`exit 3`, `pnpm dev`, `HEAD`).
